# 系譜会計と前提空間 (Fable5 案C / 案2)

## 案C: 認識の系譜会計 — 自己参照汚染の遮断

### 名指しする問題

**KIK-466（過去情報と現在情報の統合解釈）は看板機能であると同時に汚染経路である。**

レポートが Neo4j に入り、次のレポートが過去レポートを解釈に使い、その解釈がまた保存される。
時間とともにシステムの「知識」に占める自己生成物の比率が単調増加し、一次観測由来の主張と
自分の過去の推論由来の主張が同格で混入する。

plan-execute の複数エージェントも同じ記憶を共有するため、**その合意は独立検証ではなく
同一汚染源の反響になり得る。** 記憶が優秀であるほど、運用が長いほど悪化する。

### 四系譜（排他）

| 系譜 | 深度 | 例 |
|:---|---:|:---|
| `primary_observation` 一次観測 | 0 | 価格・EDINET/TDnet/EDGAR の開示原文・IR原文 |
| `external_discourse` 外部言説 | 1 | ニュース・センチメント・アナリスト見解・**yfinance等の加工済み指標** |
| `user_statement` ユーザー言明 | 1 | テーゼ・投資メモ |
| `self_inference` 自己推論 | 祖先の最大+1 | システム自身の解釈 ← **汚染源** |
| `legacy` | 1 | 案C導入前の既存ノード。遡及型付けはしない |

**型付けは取得元ドメインで機械判定し、自己申告を認めない。** 又聞きニュースを一次情報と
偽装されると深度会計そのものが無意味になるため。出所不明は一次に格上げせず外部言説にする。

### 深度閾値（二段）

| 対象 | 閾値 | 挙動 |
|:---|---:|:---|
| 保有銘柄 | 深度3以上 | 再接地なしに新しい解釈へ使えない |
| ウォッチ | 深度4以上 | 同上（緩和） |

再接地の錨は **一次観測でなければならない**。ニュースへの接地は再接地ではない。

再接地は「現在アクティブな解釈に使われる主張のみ」を対象にする怠惰評価。
全 legacy 資産の再導出という非現実的なコストを避けるため。

### レポートへの常設

解釈を出力するときは末尾に系譜サマリを付ける:

```
本解釈の根拠: 一次観測62%／外部言説21%／自己推論17%（最大深度2）
```

自己推論が過半、または深度が閾値超えなら:

```
⚠️ 本見解は当システムの過去解釈への依存が強く、独立根拠が薄い。
```

### plan-execute / plan-check への影響

- **Devil's Advocate の攻撃対象に導出鎖を追加する。**
  「この前提の根拠は3世代前の自システムのレポートのみ」という指摘ができる。
- エージェント間で**系譜の異なる証拠を意図的に分配**し、同一汚染源による
  疑似合意を構造的に防ぐ。

### 案B との接続

`claims_from_decision_package()` が DecisionPackage を Claim に変換する。
可知集合の `used` が根拠（開示時刻があれば一次観測）、結論が自己推論になる。
**案B を Claim の最初の供給源にすることで、系譜台帳を二重実装せずに立ち上げる。**

### 非破壊

legacy ノードの削除・変換は禁止。型付け義務は新規書込のみ。
汚染度は**警告であり、レポート出力自体は止めない。**

## 案2: 前提空間ダイバーシフィケーション

### 名指しする問題

分散はセクター・地域・通貨・規模の HHI で測られてきたが、これは**資産空間**の集中度にすぎない。
thesis に書かれた「なぜ持つか」が複数銘柄で同一前提（円安継続・AI設備投資・金利低下）に
依存していても、既存のどの指標にも映らない。

**価格相関は前提相関の遅行指標である。** セクター分散済みのPFが、単一の仮定の反転で
全滅するリスクが不可視のまま残る。

### 前提HHI

thesis ノートから前提を抽出し、既存の `compute_hhi` をそのまま使って前提空間の集中度を測る。

| 水準 | 閾値 | 意味 |
|:---|---:|:---|
| ok | < 0.35 | 前提は分散している |
| warning | ≥ 0.35 | 特定の前提への依存が大きい |
| danger | ≥ 0.50 | 一つの物語に全張り。セクターが分散していてもこれは分散していない |

1銘柄が複数の前提に依存する場合、評価額は各前提に**重複計上する**。
「この前提が壊れたらいくら分が影響を受けるか」を見たいので、合計は100%を超えてよい。

前提が抽出できない場合は `unknown` を返す。**「分散している」と誤読させない。**

### 自動生成シナリオ

「あなたの最大共有前提が反転した世界」を、固定8シナリオとは**別枠**で生成する。
既存 `SCENARIOS` と同じ形（`name` / `trigger` / `base_shock` / `effects`）なので
既存のシナリオ分析にそのまま流せる。

リスクモデルの一部が、外生（一般シナリオ）から内生（自分の信念グラフ由来）に変わる。

**影響度の定義を持たない前提はスキップする。** 推測で数字を捏造しない。

### 使い方

```python
from src.core.risk.assumptions import analyze_assumption_space
result = analyze_assumption_space(holdings=[{"symbol": "SOXL", "value": 6800000}, ...])
result["concentration"]["message"]   # 前提HHIの評価
result["scenarios"]                  # 反転シナリオ（最大3件）
```

thesis が溜まるほど精度が上がる。**投資理由をメモに残すことが、そのまま
リスクモデルの入力になる。**
