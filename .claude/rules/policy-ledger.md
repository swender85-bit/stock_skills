# Policy Ledger: 判断と執行の時間分離 (Fable5 第2弾 案A)

## なぜこれがあるか

ユーザーが質問する瞬間は、市場が動いて感情が動いた瞬間と強く相関する（下落時の「売るべき？」）。
つまり従来のシステムは、**判断の質が最も劣化する時刻に判断を生成する構造**を持っていた。

政策台帳は、冷静な時点で「状態→応答」を確定しておき、事態成立時は政策を機械的に参照する。
出力の型が「行為の推奨」から「政策の参照」に変わる。**新しく判断しないことが正当な出力になる。**

## 意思決定状態

従来の 買う／売る／保有 の3種に加えて:

| intent | 意味 |
|:---|:---|
| `conditional_commit` | 条件付きコミット（この条件になったらこうする） |
| `awaiting_trigger` | トリガー待機（監視のみ、成立まで動かない） |
| `deliberate_inaction` | 意図的不作為（「見ない／動かない」を選ぶ宣言） |

`deliberate_inaction` は重要。「何もしない」を明示的な意思決定として記録できると、
後から「決めた通りに動けたか」を測れる。

## 運用ルール

### 急変時（最優先）

```bash
python scripts/manage_policy.py ask --symbol 2802.T --state "drawdown_pct=-6"
```

出力される「政策上の応答」を**分析より先に**提示する。
政策が無ければ従来どおり分析してよい。

### 登録

```bash
python scripts/manage_policy.py add --symbol 2802.T \
    --response "全株売却" --trigger "drawdown_pct<=-25" \
    --expires 2027-01-31 --rationale "本命は営業CF悪化。価格は補助条件"
```

- **失効期限は必須**。無期限政策は登録できない。期限切れ＝再審査の強制。
- トリガーは測定可能でなければ拒否される。「なんか下がったら」は登録できない。
- 同一指標の矛盾条件（`x<=-25` かつ `x>=-5`）も拒否される。

使える指標: `price` / `price_change_pct` / `drawdown_pct` / `rsi` / `per` / `pbr` /
`dividend_yield` / `operating_cf` / `operating_margin` / `position_weight_pct` / `days_held`

### 改訂

**平時の改訂は自由。トリガー接近・成立中の改訂だけ24時間の冷却期間がかかる。**

これは意図的な非対称性で、平時の柔軟性（環境変化への追随）と
有事の拘束（狼狽による政策破棄の防止）を分離するための設計。

### 逸脱

政策と実際の売買記録が食い違ったら逸脱として記録される。2種類:

- `acted_without_trigger` — トリガー未成立なのに政策と反する売買をした
- `ignored_trigger` — トリガー成立済みなのに政策の応答を実行していない

**逸脱は「過程」の誤りなので、案Bの二層化で `origin=process` になり、
plan-check の制約になる資格を持つ。** 結果由来の lesson とは扱いが違う。

## 副作用と抑制策

| 副作用 | 抑制策 |
|:---|:---|
| 政策の硬直化 | 失効期限の必須化 + 期限切れ時の再審査強制 |
| ストレス下の政策破棄 | トリガー接近・成立時のみ冷却期間 |
| 曖昧な政策の氾濫 | 登録時の実行可能性審査（測定可能性・矛盾検出） |
| 「政策があるから考えない」 | 期限ごとの再審査で必ず思考を再起動させる |

## 非破壊条件

- 政策未登録の銘柄には従来動作を完全維持する。
- 既存 watchlist データは変換せず併存する。
- 政策層は全面 opt-in。
